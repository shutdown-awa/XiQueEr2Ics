## 添加自己的学校

如果您希望为本项目添加新的学校支持，可以按照以下步骤操作（**不保证 100% 成功**）：

> 💡 提示：您可以尝试复制其他学校的配置文件进行修改，通常情况下各学校的喜鹊儿接口差异不大。

1. 在 `/schools` 目录下创建以学校代码命名的文件夹（**纯数字**）。  
   - 如果您计划提交 PR，请使用可在百度搜索到的**学校代码**作为文件夹名。

2. 复制其他学校的文件夹内容作为模板。

3. 修改 `/schools/your_school_code/config.json`，使其符合您学校的实际情况。

4. 修改 `/schools/your_school_code/timetable.json`，配置上下课时间。  
   > 说明：未从教务系统自动拉取作息时间，是因为开发者所在学校的作息表经常不准，因此改为手动配置。

5. 运行 `maintain.py`，脚本会自动拉取**过去 5 年内（包含当前年份）所有学期**的开学时间和假期时间。  
   - ✅ 请务必在**每个学期开始前**重新运行一次此脚本。  
   - ✅ 请确保在完成第 3 步后再执行此步骤。

6. 如果您希望搭建 Web 服务以方便生成订阅链接，请修改 `/web/school.json`，并按照上述步骤添加您的学校信息。

## 自行搭建服务

### Docker Compose（推荐）

1. 创建 `user` 目录（用于持久化用户缓存数据）：

   ```bash
   mkdir -p user
   ```

2. 根据需要修改 `docker-compose.yml` 中的环境变量（详见下方 [环境变量配置](#环境变量配置)）。

3. 启动服务：

   ```bash
   docker compose up -d
   ```

4. 服务默认运行在 `http://localhost:8080`。如需自定义端口或路径，请参考环境变量说明。

> ⚠️ `user` 目录需要挂载到容器内 `/app/user`，否则容器重启后用户缓存将丢失。

### 手动部署

以下步骤可能因环境不同而有所差异，**不一定全部需要执行**，仅供参考：

#### 环境要求

- Python 环境
- Node.js 环境

#### 配置步骤

1. 修改 `/web/scripts.js` 顶部内容，配置正确的 API 域名与路径。
2. 检查并替换代码中所有包含 `"hishutdown.cn"` 域名的地方，改为您自己的域名或 IP。
3. 安装 Python 依赖：

   ```bash
   pip install -r requirements.txt
   ```

4. 通过 `run-web.sh` 启动服务（推荐）：

   ```bash
   ./run-web.sh
   ```

   或直接使用 uvicorn：

   ```bash
   uvicorn api:app --host 0.0.0.0 --port 8080
   ```

> ⚠️ 若启动失败，请优先排查环境配置问题。如确认为代码缺陷，欢迎提交 Issue。

### 环境变量配置

服务通过环境变量进行配置，所有变量均有默认值，不设置也能正常运行。

| 环境变量 | 说明 | 默认值 |
|---|---|---|
| `api_host` | 监听地址 | `127.0.0.1` |
| `web_port` | 监听端口 | `8080` |
| `root_path` | API 根路径前缀（适用于反向代理场景） | `""`（无前缀） |
| `DEBUG` | 开启调试日志（设为任意非空值） | 关闭（INFO 级别） |

**使用示例：**

docker-compose：

```yaml
environment:
  - api_host=0.0.0.0
  - web_port=8080
  - root_path=/xqe2ics
```

手动运行：

```bash
api_host=0.0.0.0 web_port=3000 root_path=/xqe2ics ./run-web.sh
```

> 💡 `root_path` 适用于将服务部署在反向代理的子路径下的场景，例如 Nginx 代理到 `https://example.com/xqe2ics/` 时，应设置 `root_path=/xqe2ics`。
